# 技术架构

> 描述 apicase 的实现架构现状（v0.1）。

## 技术栈

- **桌面框架**：Tauri 2.x（Rust 后端 + 系统 WebView 前端）。
- **后端**：Rust；`reqwest`（rustls TLS，避免本地 OpenSSL）；`serde` / `serde_json`；`tauri-plugin-dialog`（目录选择）；`notify`（fs 监听）；`portable-pty`（伪终端）；`tokio`（仅测试）。
- **前端**：React 19 + TypeScript + Vite 7；`@tauri-apps/plugin-dialog`；`js-yaml`（case 解析/序列化）；`@xterm/xterm` + `@xterm/addon-fit`（底部终端）。
- **前后端通信**：Tauri command + `invoke`。
- **权限**（`capabilities/default.json`）：`core:default`、`opener:default`、`dialog:default`、`core:window:allow-start-dragging`（自定义标题栏拖动窗口需显式授权）。

## 后端：核心命令与数据模型

数据模型（`serde`，对前端以 camelCase 序列化）：

```rust
struct HeaderEntry { key: String, value: String }
struct ApiRequest  { method, url, headers: Vec<HeaderEntry>, body: Option<String> }
struct ApiResponse { status, status_text, headers, body, elapsed_ms }
```

- **`send_request(request: ApiRequest, proxy: Option<ProxyConfig>) -> Result<ApiResponse, String>`**：Tauri 命令。
- **`perform_request`**：实际发请求逻辑，与 Tauri 解耦，便于单元 / 集成测试。
  - 由 Rust 后端用 `reqwest` 发出，**天然绕过浏览器 CORS**。
  - 校验空 URL、非法 HTTP 方法；统计耗时（请求发出到响应体读完）。
  - **代理**（`ProxyConfig { mode, url? }`，前端「设置 → 代理」）：`none`→`no_proxy()` 直连、`custom`→`Proxy::all(url)`、`system`/缺省→ reqwest 默认（读 `HTTP(S)_PROXY` 环境变量）。前端偏好持久化 localStorage `apicase.proxy.v1`。
- **`init_workspace(path) -> Result<(), String>`**：把目录初始化为工作空间——校验目录存在，无 `application.yml` 则写入初始模板（含 `environment`）。幂等：有则不动，故「打开」与「创建」共用此命令。
- **应用设置**（存**应用配置目录** `settings.json`，仅按 `identifier` 定位，跨 dev / 打包一致——不像 localStorage 按 origin 分桶）：
  - `read_app_settings() -> String` / `write_app_settings(content)`：读 / 写整份 `settings.json`（原始 JSON 文本，结构由前端 `settings.ts` 定义；当前存**最近打开的工作空间**）。目录经 `app_config_dir()` 定位（macOS `~/Library/Application Support/com.apicase.app/`），文件缺失读返回空串、写前 `create_dir_all` 兜底。
- **`list_dir(path) -> Result<Vec<DirEntry>, String>`**：列目录直接子项（文件树懒加载用）。`DirEntry { name, path, isDir }`；跳过隐藏项（`.` 开头），目录在前、组内按名称（忽略大小写）排序。
- **通用文件命令**（case 读写与文件管理；YAML 解析在前端）：
  - `read_text_file(path) -> String` / `write_text_file(path, content)`：读/写文本（打开、保存 case）。
  - `create_file(path, content)` / `create_dir(path)`：新建 case / folder（拒绝覆盖已存在）。
  - `rename_path(from, to)`：重命名 / 移动（目标存在则拒绝）。
  - `delete_path(path)`：删除（文件 `remove_file`，目录 `remove_dir_all`）。
  - `search_workspace(root, query) -> Vec<DirEntry>`：递归搜索名称匹配（不区分大小写）的文件/目录；跳过隐藏项与 `node_modules`/`target`/`dist`，结果上限 200。
  - `read_file_smart(path)`：智能读取——NUL 嗅探 + UTF-8 校验（前 64KB 含 NUL 即判为二进制、提前返回不读大文件）；二进制 / 不支持编码的文件返回标记，前端据此显示「无法在文本编辑器中显示」。
- **文件系统监听**（外部增删改的实时感知，跨平台原生：macOS FSEvents / Linux inotify / Windows ReadDirectoryChangesW，基于 `notify` crate）：
  - `watch_workspace(path)`：递归监听工作空间；事件经 **250ms 去抖**、过滤噪声（隐藏项与 `node_modules`/`target`/`dist`）后，以受影响路径列表通过 **`workspace:fs-change` 事件**发往前端。再次调用替换旧监听（切换工作空间）；watcher 存于托管状态 `WatchState`，drop 即停。
  - `path_exists(path) -> bool`：外部删除检测用。
- **伪终端**（底部终端栏，基于 `portable-pty`，跨平台起系统 shell；托管状态 `PtyState = Mutex<HashMap<id, PtySession>>`）：
  - `terminal_open(id, cwd, cols, rows)`：创建 PTY 对，按 `$SHELL`（Windows 用 `COMSPEC`）在 `cwd` 起交互 shell（`TERM=xterm-256color`）；后台线程阻塞读 master，成块以 **`terminal://data/{id}` 事件**（原始 `Vec<u8>`）发往前端，进程结束发 **`terminal://exit/{id}`**。
  - `terminal_write(id, data)` / `terminal_resize(id, cols, rows)` / `terminal_close(id)`：写入用户输入 / 同步行列 / 杀子进程并释放。

## 前端：交互

请求表单 → `invoke("send_request", { request })` → 后端发请求 → 渲染响应。
UI 对标 Postman：方法下拉（彩色）与 URL **连体** + Send；Headers（自动增行）/ Body 配置 Tabs；响应区状态徽章 + 耗时 + 大小，Body(Pretty/Raw) / Headers Tabs。

**三栏布局（仿 VSCode）**：顶栏**最右**三个切换按钮（VSCode 风格图标：外框方块 + 对应边填充）显隐三块面板——**左**文件树 / **底**终端 / **右** AI 对话。布局包裹为 `.body-layout`（横向：`.sidebar` + `.center-col` + `.ai-panel`），`.center-col` 内纵向排 `.workspace` 与 `.bottom-panel`。显隐状态 `LayoutFlags{left,bottom,right}` 持久化 `localStorage`（`apicase.layout.v1`，默认左开、底/右关）；底部栏可拖高、右侧栏可拖宽（复用既有 resizer 模式，`body.resizing-row`/`resizing-col`）。底部终端与右侧 AI **首次打开后常驻**（`display:none` 隐藏而非卸载），保留 shell 会话/滚动与对话历史。快捷键（`shortcuts.ts` 「视图」组）：`Mod+B` 左栏 / `Mod+J` 终端 / `Mod+Alt+B` 右栏。
- **终端**（`TerminalPane.tsx`）：`xterm.js` + `FitAddon`；挂载即 `terminal_open`（cwd=工作空间），监听 `terminal://data/{id}` → `term.write(Uint8Array)`（原始字节交 xterm 缓冲，天然处理跨包 UTF-8）；`onData` → `terminal_write`；`ResizeObserver` → fit + `terminal_resize`；切回可见时 refit + focus；`cwd` 变化（切工作空间）重建会话。
- **AI 对话**（`AiChat.tsx`，本期 UI 外壳）：消息气泡列表（自动滚底）+ 多行输入（Enter 发送 / Shift+Enter 换行）+ 清空；发送后追加用户消息与占位助手回复，后续接真实模型仅替换回复来源。

**工作空间 / 文件树**：左上角自定义下拉 → `plugin-dialog` 选目录 → `invoke("init_workspace")` → `invoke("list_dir")` 渲染左侧文件树（懒加载、可展开折叠）；左侧栏可拖动调宽、可整体显隐；右键菜单新建 case/folder、重命名、删除。打开工作空间即 `invoke("watch_workspace")` 启动监听，订阅 `workspace:fs-change` 事件做**自愈刷新**（仅刷新已加载目录，保持懒加载一致）。

**case 读写**（`case.ts`）：点击 `.yml`（非 `application.yml`）→ `read_text_file` → `analyzeCase` → 渲染。编辑标记 dirty → `Cmd/Ctrl+S` 或保存按钮 → `dumpCase` → `write_text_file`。YAML 在前端用 `js-yaml` 解析；序列化裁剪默认值（`enabled:true`、空 body 等不落盘）。

**三视图与 flow 编辑**（`App.tsx` + `FlowCanvas.tsx` + `RequestEditor.tsx` + `draft.ts`）：case 内部统一为 step 列表模型（单节点 = 1 个 step；YAML 写 `steps:` 列表，每步含 `protocol:` 与 `request:`）。顶栏三视图切换——**文本**（原始 YAML，互斥）/ **流程**（DAG 画布）/ **请求**（复用请求编辑器），按**意图式**切换（关掉唯一在显栏落文本、从文本点某栏则确保显示该栏）。内容驱动默认视图（`analyzeCase` 判 valid/单/多）；切结构视图前 `commitText` 重解析 rawText，切文本前从结构态 `dumpCase` 刷新（结构是切换时的真相源）。画布 `dependsOn` 自动分层布局 + SVG 贝塞尔连线。**画布交互**（`FlowCanvas.tsx`）：平移（拖背景 / 滚轮）+ 缩放（Ctrl/⌘ 滚轮锚定光标，工具栏放大/缩小/缩放%复位）+ 适应视图 + 规整（清手动坐标恢复自动布局）；**拖拽节点**持久化坐标到 `ui.nodes`；从 **out 端口拖到目标节点建立依赖**（防自连/重复/成环）；边悬停显示删除按钮解除依赖；双击空白快速加节点；**拖拽对齐吸附**（与其它节点列/行对齐时贴齐并显示参考线）；右下角 **小地图/概览**（节点缩略 + 视口框，点击/拖拽导航）。坐标是视图态、拓扑（`dependsOn`）是真相。

**多标签页**（`TabBar` + `tabOrder` + `tabCacheRef`）：活动标签用 live state，非活动标签用 `TabSnapshot` 快照保存完整编辑态；`openTab` 快照当前 + 恢复目标（新标签走 `openCase` 加载），`closeTab`/`closeOtherTabs`/`closeAllTabs`（dirty 关闭前确认）；重命名/删除文件同步标签。

**flow 执行引擎**（`flow.ts`）：`发送` / `▶ 运行`（拓扑序串行）——每步**变量透传（`{{vars}}` / `{{steps.x.outputs.y}}`）→ `send_request` → JSONPath 提取 `outputs` → 评估断言**；outputs 在步骤间透传（运行全部用本地上下文在 `await` 间同步传递）。断言结果逐条 ✓/✗，失败标红节点。请求编辑器在参数/请求头/认证/请求体之外，追加 输出 / 断言 / 文档 / 请求 ID 等 Tab（输出/请求 ID 仅多请求场景，请求 ID 置于末尾；「文档」标签支持 markdown 编辑与预览，内容存 step 的 `docs:`）。后端仍只做 `send_request`，编排语义全在前端。

**environment（多套环境）**：打开工作空间读根 `application.yml` 的 `environment` 段（`parseEnvironments`），topbar 右侧下拉切换；运行时 `ctx.vars = { ...活动环境, ...case级vars }`（case 覆盖环境）。任意 `.yml`（含 `application.yml`）可在编辑器**以文本打开**——非 case 落文本视图；保存 `application.yml` 后即时重载环境。**文本视图忠实**：有效 case 无结构化改动时保留原文件文本（含注释），仅有编辑时才 `dumpCase` 重生成。

**文件树工具栏**：顶部搜索框（输入即 `search_workspace` 递归搜索，结果切换为扁平列表、点击打开）+ `+` 按钮（弹「新建 case / 新建 folder」菜单）。**可视化新建 case**：对话框填 名称 + method + URL → 生成 case 文件并自动打开；右键菜单的「新建 case」走同一对话框。

**全局快捷键**（`shortcuts.ts` + `App.tsx`）：app 级快捷键，单一 `document` keydown 分发（`ref` 取最新绑定 / 动作闭包，复用 `saveRef` 模式）。核心动作：新建用例 / 打开工作空间 / 保存 / 关闭标签 / 搜索 / 打开配置页 / 发送请求。修饰键统一抽象 `Mod`（macOS=⌘、其它=Ctrl），跨平台一套配置；纯键在输入框内不拦截（避免干扰打字），带 Mod 的组合处处生效。配置页「快捷键」分区支持查看、录制重绑、冲突检测、禁用、恢复默认；override 持久化到 `localStorage`（`apicase.shortcuts.v1`，app 级偏好，区别于 workspace 级 `application.yml`）。

**自定义标题栏**（`tauri.conf.json` 窗口配置，macOS）：
- `titleBarStyle: "Overlay"` + `hiddenTitle: true`：内容延伸进标题栏、隐藏标题文字。
- `trafficLightPosition { x, y }`：把红绿灯下移与顶栏内容垂直居中对齐（近似 `红绿灯中心 ≈ y + 7`；当前 `x:16, y:16`，顶栏高 38px）。**改此项须重启 `tauri dev`**。
- 全屏自适应：靠窗口尺寸**变化方向**判断（缩小=退出全屏，立即恢复留白，早于红绿灯出现），避免退出全屏时红绿灯覆盖下拉。

## 当前实现边界

- 可**打开工作空间、浏览文件树、读写单/多节点 case**（点击渲染、编辑保存、新建/重命名/删除）。
- **多节点 flow** 已可**可视化编辑 + 执行**（DAG 画布支持平移/缩放/适应/规整、拖拽节点、端口连线建依赖（防环）、悬停删边 + 依赖/输出/断言 + 三视图；运行时 `{{steps.x.outputs.y}}` 变量透传、JSONPath 提取、断言评估）。
- **environment**（`application.yml` 多套环境）已实现读取 + 切换 + 注入；变量**可视化编辑**尚未做（当前经文本编辑 application.yml），未定义 `{{var}}` 发送时保留字面量。
- 最近打开的工作空间**持久化**到应用配置目录 `settings.json`（`app_config_dir`，跨 dev / 打包一致，重启保留；见 `settings.ts` + `read_app_settings`/`write_app_settings`）；上限 10 条，加载时 `filterExistingPaths`（复用 `path_exists`）**剔除已删除 / 移动的失效目录**，过滤结果整份覆盖回 `settings.json`（非仅显示层过滤，文件不累积失效项）；点击最近项若目录在点击这一刻已被删除 / 移动（TOCTOU），`selectWorkspace` 先 `pathExists` 校验——失效则移除 + 提示、不切过去。下拉里每条最近项 hover 出现删除按钮可**手动移除**（仅删记录、不删目录，同步落盘）。**文件树随外部变更自动刷新**（`notify` fs watch）：外部创建/修改/删除会实时反映到文件树，外部删除的打开标签自动关闭，活动文件被外部修改时——净态自动重载、有未保存改动则弹「重新加载 / 忽略」提示（绝不静默覆盖）；`application.yml` 外部变更自动重载环境。自身写操作经「回声抑制」避免误触发。
- **快捷键**为 app 级偏好，持久化 `localStorage`（`apicase.shortcuts.v1`）；配置页可自定义重绑（录制 / 冲突检测 / 禁用 / 恢复默认）；新增「视图」组切换三面板。
- **三栏布局**：顶栏最右切换左（文件树）/ 底（终端）/ 右（AI 对话）；显隐持久化 `localStorage`（`apicase.layout.v1`）。**底部终端为真·PTY**（`portable-pty` + `xterm.js`），功能效果对标 VSCode 内置终端。**右侧 AI 对话本期为 UI 外壳**（消息列表 + 输入交互，回复占位；接真实模型待后续）。
- flow 执行为**拓扑序串行**（未并发）；响应按文本处理，大 / 二进制响应、流式留后续。
- **YAML 解析在前端**（`js-yaml`）：本阶段 case 仅渲染到 UI + 发单请求，前端解析免 round-trip、迭代快、schema 单点在 `case.ts`。后端 `serde` 统一模型推迟到 flow **执行引擎**阶段（届时调度 DAG 才需后端理解 case 结构）。
