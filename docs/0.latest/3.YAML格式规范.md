# case YAML 格式规范

> 定义 apicase 的 case 文件（`.yml`）与工作空间配置（`application.yml`）的 YAML 格式。
> **schema 单一事实源在 `app/src/case.ts`**（`parseCase` / `dumpCase`）；本文与其保持一致。

## 总览

一个 `.yml` 文件即一个 case，模型上是一个 **DAG**：单请求 = 单节点 DAG，多请求 = 多节点 DAG（节点间以 `dependsOn` 声明依赖）。写盘时**恒定使用 `steps:` 列表**（单节点也是长度为 1 的列表）；每个 step 用 `protocol:` 显式声明协议，报文承载于 `request:`。

```yaml
apicase: "0.1"          # 版本标识（必填，缺省按 "0.1")
name: 用户登录并下单       # 可选：case 名称
vars:                   # 可选：case 级变量（覆盖 environment）
  baseUrl: https://api.example.com
steps:                  # 必填：至少 1 个 step
  - id: login
    protocol: http      # 协议标识（当前仅 http）
    request:            # 报文（随 protocol 而定；http 见下）
      method: POST
      url: "{{baseUrl}}/login"
      body:
        type: json
        json: { username: admin, password: "123456" }
    outputs:
      token: $.data.token
    assertions:
      - { target: status, op: eq, value: "200" }
    docs: |             # 可选：该 step 的 markdown 文档
      登录接口，成功后从响应体提取 token 供下游使用。
  - id: createOrder
    protocol: http
    dependsOn: [login]
    request:
      method: POST
      url: "{{baseUrl}}/orders"
      headers:
        - { name: Authorization, value: "Bearer {{steps.login.outputs.token}}" }
      body:
        type: json
        json: { sku: A-1001, qty: 2 }
    assertions:
      - { target: $.code, op: eq, value: "0" }
ui:                     # 可选：画布节点坐标（可视化布局，未持久化拖拽时可缺省）
  nodes:
    login: { x: 40, y: 40 }
    createOrder: { x: 40, y: 200 }
```

## 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `apicase` | string | 是 | 格式版本，当前 `"0.1"`。 |
| `name` | string | 否 | case 显示名。 |
| `vars` | map | 否 | case 级变量；优先级高于 environment。 |
| `steps` | list | 是 | step 列表，至少 1 个。 |
| `ui.nodes` | map | 否 | 画布节点坐标 `{ 请求id: {x, y} }`。 |

## step（请求节点）

字段书写顺序（与 `dumpCase` 对齐）：`id → protocol → dependsOn → request → outputs → assertions → docs`。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | 是 | 节点唯一标识，被 `dependsOn` 与 `{{steps.<id>...}}` 引用；改名需同步引用。 |
| `protocol` | string | 是 | 协议标识，当前仅 `http`；缺省按 `http`。 |
| `dependsOn` | string[] | 否 | 上游依赖的 step `id`；据此自动分层布局与拓扑序执行。空则省略。 |
| `request` | object | 是 | 报文（随 `protocol` 而定）；`http` 报文见下。 |
| `outputs` | list | 否 | 从响应体提取变量供下游引用。空则省略。 |
| `assertions` | list | 否 | 响应断言。空则省略。 |
| `docs` | string | 否 | 该 step 的 markdown 文档（请求编辑器「文档」标签）。空则省略。 |

### request（http 报文）

`protocol: http` 时 `request` 的字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `method` | string | GET / POST / PUT / DELETE… |
| `url` | string | 请求 URL，可含 `{{var}}`；query 与 url 双向同步。 |
| `query` | KV[] | 查询参数（UI「参数」页），空则省略。 |
| `headers` | KV[] | 请求头，空则省略。 |
| `auth` | object | 认证，`type: none` 时省略。 |
| `body` | object | 请求体，`type: none` 或空时省略。 |

**KV** 结构：`{ name, value, enabled? }`。`enabled` 默认 `true` 且省略，仅关闭某行时写 `enabled: false`。

### auth（认证）

`type` 取值：`none` / `bearer` / `basic` / `apikey`，对应子对象二选一：

```yaml
auth: { type: bearer, bearer: { token: "{{token}}" } }
auth: { type: basic,  basic:  { username: admin, password: "123456" } }
auth: { type: apikey, apikey: { key: X-Api-Key, value: "{{apiKey}}", in: header } }  # in: header | query
```

### body（请求体）

`type` 取值：`none` / `json` / `text` / `form-urlencoded` / `form-data`。

```yaml
body: { type: json, json: { a: 1, b: "x" } }              # 结构化对象，diff 友好
body: { type: text, text: "raw...", contentType: text/xml } # contentType 可选，覆盖 Content-Type
body: { type: form-urlencoded, urlencoded: [ { name: a, value: "1" } ] }
body: { type: form-data, formData: [ { name: a, value: "1" } ] }  # 当前仅 text 字段，file 后续
```

### outputs（输出提取）

map 结构 `{ 变量名: JSONPath }`：键为变量名，值为 JSONPath（常用子集），从响应体提取供下游 `{{steps.<id>.outputs.<变量名>}}` 引用。

```yaml
outputs:
  token: $.data.token
  orderId: $.data.id
```

### assertions（断言）

列表项结构 `{ target, op, value? }`：

- **`target`**：断言目标 —— `status`、`header.<名>`、或 JSONPath（如 `$.code`）。
- **`op`**：`eq` / `ne` / `contains` / `exists` / `notExists` / `gt` / `lt` / `matches`。
- **`value`**：期望值；`exists` / `notExists` 无需 value（省略）。

```yaml
assertions:
  - { target: status, op: eq, value: "200" }
  - { target: header.Content-Type, op: contains, value: application/json }
  - { target: $.data.token, op: exists }
```

### docs（step 文档）

`docs` 为一段 markdown 文本，在请求编辑器「文档」标签中编辑与预览（支持标题、列表、代码块、表格、引用、链接、图片等）。空则不落盘。

## 变量与透传

- **语法**：`{{name}}`；跨节点引用上游输出用 `{{steps.<step id>.outputs.<输出名>}}`。
- **来源与优先级**（就近覆盖）：environment（`application.yml`）< case 级 `vars` < 上游节点 `outputs`。
- 未解析到的 `{{var}}` **保留字面量**（发送时不报错）。

## 序列化约定（dumpCase）

- 恒定写 `steps:` 列表（单节点 = 长度 1）；**读取仅认此格式** —— 旧格式（`requests:` 列表、`http:` 报文键）不再兼容，可视化编辑器会将其判为无效 case 并回退为纯文本编辑。
- 裁剪默认值：`enabled: true`、空 `query/headers/outputs/assertions/dependsOn`、`body.type=none`、`exists/notExists` 的 `value`、空 `docs` 均不落盘。
- `json` body 以结构化对象存储（便于 diff），而非转义字符串。

## application.yml（工作空间配置）

工作空间根目录的配置文件，含多套 environment：

```yaml
environment:
  dev:  { baseUrl: https://dev.example.com,  token: "" }
  prod: { baseUrl: https://api.example.com,  token: "" }
```

topbar 右侧下拉切换活动环境，运行时注入变量（值统一转字符串）。详见 [2.技术架构](2.技术架构.md)。
