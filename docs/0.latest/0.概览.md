# apicase 概览

> 本目录（`docs/0.latest/`）是项目**当前全局最新的全部文档**，事实来源。
> 需求落地后将要点合并至此，保持其始终反映最新状态。

## 是什么

apicase 是一款 **API 接口调试、管理与用例编排** 的桌面软件。核心抽象：把「用例（case）」建模为 **DAG**——单个 API 调试 = 只有一个节点的 DAG，单 API 与编排用例在模型上统一。

设计上走 **文件即数据（local-first）** 路线：打开一个目录即一个**工作空间（workspace）**，folder 即分组，case 即用例。参考标杆 **Bruno**（理念）与 **Postman**（UI / 功能）。

## 当前能力（v0.1）

**脚手架 + 单 API 调试 MVP** —— 已落地：
- Tauri 2 桌面应用骨架可编译运行。
- 界面发起一个 HTTP 请求 → 由 Rust 后端（`reqwest`）发出 → 展示响应（状态码 / 耗时 / 大小 / 响应头 / 响应体）。即「单节点 DAG」的端到端执行。
- Postman 风格 UI：方法下拉与 URL **连体** + Send 请求行；Headers / Body 配置 Tabs；响应区 Body(Pretty/Raw) / Headers Tabs，且与请求编辑区上下可拖动分栏、可折叠为一行。
- 错误处理：空 URL、非法方法、网络失败均有可读提示。

**工作空间与界面外壳** —— 已落地（2026-06-30）：
- 左上角「选择工作空间」自定义下拉：打开/创建工作空间（选目录 → 幂等写 `application.yml`）、最近列表（内存态）。
- 左侧**文件树**：选中工作空间后懒加载、可展开/折叠目录；左侧栏可拖动调宽；空状态可点击「创建或打开」。
- macOS **自定义标题栏**：隐藏标题、红绿灯下移与下拉垂直对齐、顶栏可拖动窗口；全屏时下拉贴左、退出全屏平滑让位不覆盖。

**case 读写与格式** —— 已落地（2026-06-30）：
- 点击 `.yml` case → 解析渲染到请求构建器；编辑后 `Cmd/Ctrl+S` 写回；文件树右键新建/重命名/删除 case 与 folder。
- 文件树顶部**搜索栏**（递归搜全工作空间）+ **`+` 新建按钮**；新建 case 走**可视化对话框**（名称 + method + URL），创建即打开。
- 请求构建器对齐 Postman：**Params / Headers / Auth / Body** 四 Tab（Params 与 URL 双向同步，Body 多类型，Auth 含 bearer/basic/apikey）。
- **case YAML 格式定稿**（参考业界）：request 借 Postman/HAR/Insomnia/Bruno，flow 借 Arazzo（OpenAPI 官方 workflow）+ GitHub Actions；单/多节点统一，详见概念模型。

**flow DAG 画布与三视图** —— 已落地（2026-06-30）：
- case 顶栏**两级视图切换（文本 | 可视）**：文本 = 原始 YAML（互斥）；点「可视」展开二级 **流程 / 请求**（流程 = DAG 画布、请求 = 复用编辑器，二者可并存、至少保留其一）。
- **内容驱动默认视图**（仅打开时决定）：校验不通过 → 文本兜底；有效单请求 → 请求；有效多请求 → 流程 + 请求。
- **DAG 画布**：`dependsOn` 自动分层布局（支持 fan-in）、SVG 贝塞尔连线、节点卡片（method 徽章 + 运行状态点）、选中高亮、添加/删除步骤；点状网格背景。
- **步骤编排**：请求 ID / 输出 / 断言 / 文档均为请求编辑器内的 Tab（请求 ID 位于末尾，改名同步引用），输出提取用「变量名 / JSONPath」；「文档」标签支持 markdown 编辑与预览；`发送` 与 `▶ 运行`（拓扑序逐节点执行）。依赖不再手动配置——新增请求默认串接当前选中请求，DAG 连线据 `dependsOn` 自动呈现。单节点与多节点发送按钮统一为 `发送`。
- 内部统一 step 列表模型（单节点 = 1 个 step），单 → 多平滑提升；读写**仅认** `steps:` 列表（每步含 `protocol:` 与 `request:`）——旧格式（`requests:` 列表 / `http:` 报文键）不再兼容，会被判为无效 case 回退纯文本编辑。

**flow 执行引擎** —— 已落地（2026-07-01）：
- **变量透传**：运行时把 case 级 `vars` 与上游 `outputs` 注入请求各字段；语法 `{{baseUrl}}` / `{{steps.login.outputs.token}}`，未解析保留字面量。
- **输出提取**：步骤响应体按 `outputs`（JSONPath 常用子集）提取变量供下游引用。
- **断言**：每步（及单节点顶层）配置断言（`eq/ne/contains/exists/notExists/gt/lt/matches`，目标 `status` / `header.名` / JSONPath），运行后逐条 ✓/✗，失败标红节点。
- **运行全部**：拓扑序串行执行，`await` 间同步透传 outputs，数据真正在步骤间流动。

**environment 多环境 + 界面优化** —— 已落地（2026-07-01）：
- **多套环境切换**：读 `application.yml` 的 `environment` 段，topbar 右侧下拉切换（dev/test/prod…）；运行时注入变量，优先级 **case 级 `vars` > environment**。`application.yml` 可在编辑器以文本打开编辑，保存即时重载。
- **文本视图忠实展示**：有效 case 切「文本」时，无结构化改动则保留原文件文本（含注释/格式），仅在有编辑时才重新序列化。
- **文件树美化**：SVG 目录/文件图标（case 橙、配置文件后改为齿轮，齿轮配色见 2026-07-02 与 2026-07-16）、选中态左侧橙色强调条、更柔和的 hover 与间距。（选中底色后由橙改灰、强调条保留，见 2026-07-16 条目。）

**多标签页 + 视图切换修复** —— 已落地（2026-07-01）：
- **多标签页**：可同时打开多个 case，标签栏切换（活动标签白底 + 顶部橙条、dirty 点 / hover 显 ×、中键关闭）；右键「关闭当前 / 其他 / 全部标签页」；非活动标签完整保留编辑态（快照）。
- 修复三视图切换在「关请求→关流程→落文本后再点流程无响应」的状态 bug（改为意图式切换）。
- **界面文案全中文化**（用例 / 请求 / 参数 / 请求头 / 认证 / 请求体 / 响应 / 发送 / 输出 / 断言…），仅保留 HTTP 专业术语（方法名、Content-Type、Bearer、JSONPath、DAG/YAML 等）；断言操作符存储仍用英文标识、界面显示中文。后续再做国际化。
- **任意文件可打开**：非 case 文件（.json/.txt/.md 等）点击即以纯文本显示；仅 case（.yml/.yaml）文件树图标为橙色。
- **结构化渲染只由扩展名决定**：仅 `.yml`/`.yaml`（非 `application.yml`）且内容合法才渲染成结构化编辑器；`.txt`/`.json` **即使内容符合 case 格式也只当纯文本**（否则保存会用 `dumpCase` 的 YAML 覆盖原文、丢内容——尤其 JSON）。非 case 文件不显示流程/请求视图切换、也不挂"不是有效用例"提示。
- **文本/二进制判定下沉 Rust**：`read_file_smart` 命令做 NUL 嗅探 + UTF-8 校验（前 64KB 含 NUL 即二进制、提前返回不读大文件）；二进制/媒体/不支持编码仿 VSCode——打开标签但编辑区居中提示「此文件是二进制文件或使用了不受支持的文本编码，所以无法在文本编辑器中显示。」，已知二进制扩展名前端直接短路。
- **application.yml 可视设置页（仿 GitHub 设置）**：顶栏「文本 | 可视」切换；可视 = 左导航（通用 / 主题 / 环境）+ 右配置面板。**环境**页可视化增删环境、编辑变量（`KVTable`），保存写回 `application.yml`（`dumpApplicationConfig` 保留其它顶层键）；与 topbar 环境下拉、变量注入实时联动。（环境页版式后改为左列表 + 右详情，见 2026-07-16 条目。）
- **展开/折叠箭头换 SVG chevron**：文件树目录、工作空间/环境下拉的箭头改为更大更清晰的 chevron 图标，带旋转过渡。

- **配置页「环境」分区改二级布局**：由「横向环境标签页 + 说明文字」改为**左列表 + 右详情**——左列顶部为环境搜索框（按名称过滤）与 `＋` 新增按钮，下方为环境列表（选中高亮、hover 出 `×` 删除）；右侧顶部为**可改名**的环境名输入框（失焦/回车提交，重名与空名拒绝并回滚，重建对象以保持环境顺序），下方为变量表（`KVTable`，可增删行）。`＋` 新增直接建「新环境」（重名自动加序号）并选中，改名在右侧完成，不再用 `window.prompt`。环境变量无启用/停用语义，故 `KVTable` 新增 `hideEnabled` 隐藏勾选列。原说明文字（`{{变量名}}` 引用、活动环境切换、case 级 `vars` 覆盖）已移除。

- **禁用 WebView 默认右键菜单**：`main.tsx` 在 `document` 兜底拦截 `contextmenu` 默认行为，去掉「重新载入 / 检查元素 / 查询 / 翻译 / 用谷歌翻译」等浏览器菜单；应用自有右键菜单（文件树/标签页/搜索行，均走 `onContextMenu`+`setState`）不受影响。
- **快捷键功能总开关**：快捷键页标题行「全部恢复默认」左侧新增一个总开关（`.sc-switch`），控制整个快捷键系统的启用/禁用。关闭时把全局反查表置空（`scLookupRef.current = {}`），任何键都不再分发；状态经 `loadShortcutsEnabled`/`saveShortcutsEnabled` 持久化到 localStorage（`apicase.shortcuts.enabled.v1`，缺省视为启用）。关闭时下方快捷键列表整体淡出（`.sc-list.is-off`，`opacity:.5`）作停用提示，绑定仍可编辑、只是不生效。「全部恢复默认」由红色文字链接（`link-danger`）改为按钮样式（`.sc-btn.sc-reset`，hover 转红）。原顶部操作说明文字（Esc/Backspace/Mod 用法）已删除。通用页「工作空间基本信息（只读）。」说明文字亦删除。

**深色主题** —— 已落地（2026-07-17）：
- **双主题变量体系**：`App.css` 的 `:root` 扩为一套完整语义变量（骨架 `--bg`/`--panel`/`--panel-2`/`--border`/`--border-strong`/`--text`/`--text-strong`/`--muted`、交互 `--hover`/`--hover-strong`/`--selected`/`--selected-hover`、语义色、`--accent-tint*`/`--accent-ring`、阴影 `--shadow-*`/`--overlay`、提示框 `--warn/error/info-*`、代码 `--code-*`/`--inline-code-*`、画布 `--canvas-*`/`--edge`/`--dot-idle`/`--minimap-*`、AI、图标等）；新增 `:root[data-theme="dark"]` 覆盖同名变量给出深色取值。原散落全文的 **92 处硬编码颜色**（多套相近的浅灰 hover 底、两套琥珀告警、7 档 accent 半透明、8 档黑阴影等）全部收敛到变量，仅保留 2 处有色微光（危险项红、拖放绿）。
- **原生控件跟随**：`:root` 加 `color-scheme: light`、dark 加 `color-scheme: dark`，让滚动条/input/下拉/复选框自动适配；另加 `::-webkit-scrollbar` 自定义配色（走变量）。
- **主题模块与持久化**：新增 `theme.ts`（`ThemeMode = light|dark|system`，`loadThemeMode`/`saveThemeMode`/`resolveTheme`/`applyTheme`，持久化 `apicase.theme.v1`）。`main.tsx` 首帧前 `applyTheme(loadThemeMode())` 防加载闪白；`App` 顶层管 `themeMode` state，写 `documentElement.data-theme`，`system` 模式监听 `prefers-color-scheme` 变化。
- **终端与画布**：`TerminalPane` 内置 `LIGHT_THEME`/`DARK_THEME` 两套 xterm 配色，随 `theme` prop 热更新 `term.options.theme`（不重建会话）；`FlowCanvas` 箭头 marker 改 `style.fill: var(--edge)`。
- **配置页「主题」分区**：由占位文字改为三张预览卡片（浅色 / 深色 / 跟随系统，`.theme-card`+`.theme-swatch`），选中高亮 + ✓；卡片预览色块用固定色（不跟随当前主题，以真实展示各模式外观）。

**请求页 · 响应区可拖动折叠** —— 已落地（2026-07-21）：
- **响应区高度可拖 + 折叠**：请求编辑器（上）与响应区（下）改为上下两段——`.request-pane` 由整块滚动改为 `overflow:hidden`，上段 `.req-scroll` 填充剩余高度内部滚动，下段 `.response` 高度由 `respHeight`（px）控制。响应区顶部新增始终显示的「响应」标题栏（`.response-head`），兼作**上下拖动手柄**（`row-resize`，复用 `resizing-row` 光标/全局禁选）：由请求面板底边反推目标高度，钳制 `[120, 面板高−150]`（给编辑器留 ≥150）。**拖到最下（<60px）自动折叠**为一行标题（`.is-collapsed`），点击标题行或左侧 chevron 展开、恢复折叠前高度。状态码/耗时/大小从 Tab 行移入标题栏右侧（`.response-head-meta`），**折叠态也可见**；响应内容（`.response-content`）填充剩余高度、由 `.tab-panel` 内部滚动。

**设置 · 代理开关** —— 已落地（2026-07-21）：
- **代理三态设置**：设置页新增「代理」分区（跟随**应用**组，位于「主题」后，导航配地球图标）——**跟随系统 / 不使用代理 / 自定义**（http/https 地址）三张卡片（`.proxy-card`）+ custom 模式地址输入框。后端 `perform_request` 增参 `proxy: Option<ProxyConfig>`，据 `mode` 构建 reqwest client：`none`→`no_proxy()`、`custom`→`Proxy::all(url)`（空地址退化直连、非法地址报错）、`system`/缺省→默认（读 `HTTP(S)_PROXY` 环境变量）；`send_request` 命令新增 `proxy` 参数透传。前端 `proxy.ts` 管配置（`ProxyMode`/`ProxyConfig` + localStorage `apicase.proxy.v1` + `proxyPayload`），发请求时随载荷带上。**解决「系统开代理时请求本地 `127.0.0.1` 被拦成 502、无法调试本地服务」**。reqwest 仅 `rustls-tls` feature，custom 支持 http/https（socks 需另加 `socks` feature）。
