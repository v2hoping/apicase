# flow DAG 画布与三视图切换

> 需求时间：2026-06-30。在「case 读写与格式」之上，落地多节点 flow 的**可视化编辑**，并确立 case 编辑区的**三视图切换**交互，兼顾单 API 调试（Postman 手感）与 case 流程编排。

## 背景与产品决策

此前多节点 `steps:` 的 case 只识别 + 占位提示，无法编辑。核心矛盾是：**单 API 用户不该为 flow 能力付任何界面代价**，同时 flow 又要能优雅浮现。经产品讨论定案（要点）：

1. **三视图切换（文本 / 流程 / 请求）**，放在 case 顶栏右侧段控件。
   - **文本**：原始 YAML 编辑，**互斥模式**（不与结构视图并存，规避「两个真相源」同步问题，取 Bruno code-view 心智）。
   - **流程 / 请求**：结构化分栏，可并存。流程 = DAG 画布；请求 = 复用的 Postman 式请求编辑器。
2. **内容驱动默认视图**（仅在打开时决定，不随每次按键反应）：
   - 校验不通过（YAML 解析失败 / 缺 `request` 且缺 `steps`）→ **文本**兜底。
   - 有效且**单请求** → **请求**。
   - 有效且**多请求** → **流程 + 请求**（画布默认选中第一个节点）。
3. **DAG 用画布而非线性步骤轨**：`dependsOn` 允许 fan-in（多上游汇聚到一个下游），线性列表无法诚实表达，故用 2D 画布 + 自动布局。
4. **单 → 多的提升路径**：单节点 case 打开「流程」视图即见 1 节点画布，「添加步骤」即成多节点；内部统一用 steps 模型（单节点 = 1 步），保存时 `steps.length<2` 且无 outputs/dependsOn 则回写顶层 `request:`，否则写 `steps:`。

## 实现要点

### 数据层（`src/case.ts`）
- 新增 `Step`（id / request / dependsOn / outputs）、`StepOutput`、`UiNodes` 类型；`Case` 扩展 `steps` 与 `ui`。
- `parseCase` 解析 `steps`（含 `dependsOn`、`outputs` 对象→数组、`ui.nodes` 坐标）；`dumpCase` 序列化 flow（顺序 id→dependsOn→request→outputs），裁剪默认值不变。
- 新增 `analyzeCase(text)`：返回 `{ valid, case?, error? }`，用于「内容驱动默认视图 / 文本兜底」——valid 仅当能 parse 成对象且含 `request` 或 `steps`。

### 编辑态与请求编辑器复用
- `src/draft.ts`：`ReqDraft`（编辑态草稿，body 以字符串保存，允许中途非法 JSON）+ `requestToDraft` / `draftToRequest`（保存边界做 JSON 校验）+ `buildApiRequest`（组装后端 `send_request` 载荷）。
- `src/RequestEditor.tsx`：从 `App.tsx` 抽出的**通用请求编辑器**（含 `KVTable`），受控于 `ReqDraft`。单节点与 flow 的每个 step **完全复用**它，肌肉记忆不变。切换 step 时父组件用 `key` 强制重挂载。

### DAG 画布（`src/FlowCanvas.tsx`）
- **自动布局**：`dependsOn` 最长依赖链深度分层（左→右），层内按顺序排行；带环保护；有 `ui.nodes` 坐标则覆盖自动布局。
- **连线**：SVG 三次贝塞尔 + 箭头 marker；与选中节点相连的边高亮。
- **节点卡片**：method 徽章（沿用配色）+ id + 运行状态点（idle/running(脉冲)/ok/err）+ 进出端口 + hover 删除按钮；选中橙色环。
- 画布点状网格背景（n8n / Postman Flows 观感）。

### 编排（`src/App.tsx`）
- 内部统一 `StepDraft[]` + `selectedStepId`；`isFlow = 步数≥2 或 任一步有 outputs/dependsOn`。
- 视图切换：`textMode`（互斥）+ `showFlow` / `showRequest`；`effectiveText = textMode || 步数0 || 两栏皆关`。切进结构视图前 `commitText` 重解析 rawText，非法则 alert 不切换；切进文本前从结构态 `dumpCase` 刷新 rawText（结构是切换时的真相源）。
- **步骤元信息条** `StepMeta`：步骤 ID（失焦提交 + 依赖引用同步改名）、依赖（chip 多选）、输出（`KVTable` 复用，变量名 / JSONPath）。
- **运行**：`▶ 跑此步`（按节点发请求，状态叠加到画布）；`▶ 运行全部`（拓扑序逐节点执行）。**变量透传（`{{steps.x.outputs.y}}` 解析）属执行引擎，本次暂不做**，故运行为「逐节点执行」，UI 已注明。
- 保存分两态：文本态直接写 rawText（有效则回填结构态）；结构态 `dumpCase` 写回（JSON body 非法则提示不写）。
