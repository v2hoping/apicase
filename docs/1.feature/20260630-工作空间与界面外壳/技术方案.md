# 工作空间与界面外壳

> 需求时间：2026-06-30。在「单 API 调试 MVP」之上，落地**工作空间打开/创建 + 文件树**，并把窗口外壳做成 Postman 风格的**自定义标题栏**。

## 需求内容

### A. 工作空间（workspace）选择与管理
- 左上角「选择工作空间」下拉（Postman 风格自定义下拉，非原生 `<select>`）：
  - 触发按钮初始无边框无阴影，hover 变暗；未选择时显示灰色占位「选择工作空间」。
  - 下拉面板：**打开或创建工作空间**按钮 → 分割线 → 「最近」标题 → 最近列表（可滚动，显示目录名 + 灰色完整路径）。
- 打开/创建合一：选目录后调用幂等的 `init_workspace`——目录已有 `application.yml` 即「打开」，无则写入模板「创建」。
- 「最近」列表：内存态，去重、最多 10 条（重启清空，持久化待后续）。

### B. 文件树
- 左侧栏在选中工作空间后显示文件树，**懒加载**：展开目录才读其子项并缓存。
- 可展开/折叠目录（`▸/▾`），文件点击高亮（橙色浅底）；隐藏项（`.` 开头）不显示。
- 未选工作空间时显示提示「未找到工作空间. 创建或打开工作空间.」，其中「创建或打开」可点击（复用打开/创建逻辑）。
- 左侧栏可拖动调宽（手柄 `relative` + 负 margin 热区，范围 160–480px）。

### C. 自定义标题栏（macOS）
- `titleBarStyle: Overlay` + `hiddenTitle`：隐藏标题文字、网页内容延伸进标题栏，红绿灯叠在左上。
- `trafficLightPosition` 把系统红绿灯**下移到顶栏垂直中心**，与「选择工作空间」对齐（机制见下「关键决策」）。
- 顶栏 `data-tauri-drag-region` + 权限 `core:window:allow-start-dragging` → 顶栏空白区可拖动窗口。
- **全屏自适应**：全屏时红绿灯消失，下拉贴左（padding 16，瞬时）；退出全屏时下拉**平滑让位**后红绿灯才出现，不覆盖。

### D. 请求行
- method 下拉与 URL 输入**连体**（共用一个边框、中间分隔线），初始无阴影，聚焦时整体高亮（`:focus-within`）。

## 关键实现 / 决策

- **新增依赖**：前端 `@tauri-apps/plugin-dialog`；Rust `tauri-plugin-dialog`。
- **新增权限**（`capabilities/default.json`）：`dialog:default`、`core:window:allow-start-dragging`。
- **新增 Rust 命令**：
  - `init_workspace(path)`：校验目录，无 `application.yml` 则写入初始模板（含 `environment`）。
  - `list_dir(path) -> Vec<DirEntry{name,path,isDir}>`：列直接子项，跳过隐藏项，目录在前、按名（忽略大小写）排序。
- **红绿灯垂直对齐**（反复调试得到，务必记住）：
  - Overlay 模式下红绿灯位置可由 `trafficLightPosition` 自由移动——tao 在 `draw_rect` 里直接重设按钮 frame（与 titleBarStyle 无关）。
  - 对齐近似公式：**红绿灯中心 ≈ y + 7**，要居中于高度 H 的栏取 `y ≈ H/2 − 7`；`y` 每 ±1，红绿灯移 ±1px。
  - 当前定稿参数：顶栏高 **38px**、`trafficLightPosition { x:16, y:16 }`、`padding-left 86px`（左16 + 红绿灯组~54 + 间距16）、全屏时 `padding-left 16px`。
  - 改 `tauri.conf.json`（含 `trafficLightPosition`）**必须重启 `tauri dev`**；CSS/前端改动热重载即可。
- **全屏防覆盖**：不靠 `isFullscreen()`（退出时它滞后到动画结束才变 false），改用窗口尺寸**变化方向**——窗口缩小即判退出全屏，立即恢复留白（早于红绿灯出现）；放大/不变再查 `isFullscreen()` 确认进入。退出走 0.25s 平滑过渡，进入瞬时。
