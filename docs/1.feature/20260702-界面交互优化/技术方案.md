# 界面交互优化

> 需求时间：2026-07-02。一组界面交互细节打磨：① 配置文件齿轮图标 + 右上角配置入口；② 流程/请求分栏可拖动调宽；③ Tab 栏与视图切换/保存合并一行、Tab 溢出横向滚动；④ 文件树选中与活动 Tab 同步、深层文件自动展开显露。均为纯前端改动（`App.tsx` / `App.css`）。

## 一、配置文件齿轮图标 + 右上角配置入口

- **齿轮图标 `ConfigIcon`**：`application.yml` 在文件树用齿轮图标（区别于普通文件轮廓、case 橙色），一眼可辨"这是配置文件"。采用 24×24 viewBox 的居中齿轮（Material 风格），四周留约 2.5 边距，缩放渲染时齿尖不再被裁切；中心用 `fillRule="evenodd"` 镂空，行背景透出，hover/选中态不露白。颜色蓝紫 `#5b6ed6`（`.ico-config`）。
- **可复用**：`ConfigIcon({ className, size })` 支持传参，文件树与顶栏共用同一齿轮，视觉一致。
- **右上角配置按钮 `.topbar-config`**：仅在已打开工作空间时显示，位于环境选择器之后（最右侧）；点击 `openTab(application.yml)` 打开工作空间配置（与环境下拉里"编辑环境"同一入口，更顺手）。低调方形图标按钮，hover 加浅蓝底。

## 二、流程/请求分栏可拖动调宽

- 在结构化视图「流程 + 请求」并存（`.split`）时，两栏间新增可拖动分割条 `.pane-resizer`，交互/视觉参照已有的侧栏 `.sidebar-resizer`：6px 命中区、hover/拖动时中间显示 2px 橙色高亮线、拖动时全局 `col-resize` 光标 + 禁选。
- **状态**：`flowPaneWidth`（流程面板像素宽，`null` = 用 CSS 默认 44%）+ `structuredRef`（容器）+ `flowResizingRef`。`mousemove` 按 `e.clientX - 容器左边界` 计算宽度，夹取范围：流程 ≥260、尽量给请求留 ≥360（260 下限优先）。
- 拖动时流程面板套内联像素宽（覆盖 CSS 的 44%/max-width）；**双击分割条复位**（`setFlowPaneWidth(null)`）。切到单栏（仅流程/仅请求）分割条自动隐藏、恢复默认布局，切回分栏保留调好的宽度。

## 三、Tab 栏与视图切换/保存合并一行

- 文件 Tab 栏与右侧「文本｜可视 / 流程 / 请求」切换 + 「保存」合并到**同一行**，控件固定最右侧完整显示。
- **结构**：新增 `.tab-row` flex 容器接管贴边负边距与底边框；`.tab-bar` 改 `flex:1 + overflow-x:auto` —— **Tab 过多时横向滚动**，可滑动查看前后标签；右侧控件抽成 `headControls`（`.tab-controls`，`flex-shrink:0` 不参与滚动、不被压缩），从原先两个内容分支里的 `.case-head` 上移合并。
- 清理废弃的 `.case-head` / `.case-name` / `.dirty-dot` 死 CSS。

## 四、文件树选中同步 + 深层文件自动展开显露

- **单一数据源**：删除冗余的 `selectedPath` state，文件树/搜索的选中高亮直接以 `currentCasePath`（当前打开的文件）为准。这样点 Tab、关标签切邻居、关其它、打开二进制、新建用例等所有让活动文件变化的操作，高亮都会自动同步，不再分叉。
- **自动展开显露 `revealInTree(path)`**：从文件所在目录沿 `dirName` 上溯到工作空间根，收集各级中间父目录，一次性加入 `expanded`；对未加载 `children` 的目录 `loadDir` 懒加载（否则展开后子树为空、文件仍不可见）。由 `useEffect([currentCasePath, workspace])` 触发——切到折叠目录里的文件时自动展开到它。
- **滚动可见**：`TreeNode` 给选中行加 `ref`，成为选中项时（含展开后异步挂载才渲染出来的情况）`scrollIntoView({block:"nearest", inline:"nearest"})`，最小滚动、不影响横向。
