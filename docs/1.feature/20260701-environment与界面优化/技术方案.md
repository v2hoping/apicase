# environment 多环境 + 文本忠实展示 + 文件树美化

> 需求时间：2026-07-01。三项一起交付：① `application.yml` 多套环境切换并注入变量透传；② 有效 case 的文本视图忠实展示原文件；③ 文件树视觉美化。

## 一、environment 多套环境切换

**背景**：此前变量透传只覆盖 case 级 `vars` + 步骤 `outputs`，`{{baseUrl}}` 这类跨 case 公共变量无处配置。

- **数据源**：工作空间根 `application.yml` 的 `environment` 段——`{ 环境名: { 变量: 值 } }`（`init_workspace` 已写初始模板 `environment: { default: {} }`）。
- **解析**：`case.ts` 新增 `parseEnvironments(text)`，值统一转字符串；解析失败返回 `{}`。
- **切换**：topbar 右侧新增**环境下拉**（当前环境 + 变量数 + 切换 + 「编辑环境」入口）。打开工作空间时读取 `application.yml`，默认选 `default`（无则首个）。
- **注入**：运行时 `ctx.vars = { ...活动环境变量, ...case级vars }`——**case 级 vars 覆盖 environment**（更具体者优先）。`steps.x.outputs.y` 仍为独立命名空间。
- **编辑闭环**：`application.yml`（及任意非 case 的 `.yml`）现可在编辑器**以文本打开**；保存 `application.yml` 后即时重载 environment，切换生效。文本区顶部对配置文件显示专用提示条。

## 二、文本视图忠实展示

**问题**：切到「文本」时总是 `dumpCase` 重新序列化，丢失原文件注释/格式/键序。

- **修正**：`enterText` 改为 **dirty 门控**——无结构化改动时**保留打开时的原始文件文本**（含注释）；仅当有未保存的结构化编辑才 `dumpCase` 反映改动（注释不可避免地丢失）。
- 与既有「结构是切换时的真相源」一致：结构→文本仅在有编辑时才重生成。

## 三、文件树美化

- 目录/文件改用 **SVG 图标**（文件夹填充、文件描边）；case 文件橙色、`application.yml` 等非 case 文件灰色以区分。
- 行高 24→26、圆角、更柔和 hover；**选中态左侧橙色强调条** + 文名橙色加粗。
- 三角折叠符缩小转灰；根名带图标。

## 涉及文件

- `case.ts`：`parseEnvironments`。
- `App.tsx`：环境状态/下拉/加载/注入；`isYamlFile`/`isAppConfig`；打开任意 yml；保存 application.yml 重载环境；`enterText` dirty 门控；`FolderIcon`/`FileIcon` 与文件树渲染。
- `App.css`：环境下拉、文件树图标/选中条、配置提示条。
