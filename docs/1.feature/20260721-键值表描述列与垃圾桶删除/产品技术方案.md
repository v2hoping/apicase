# 键值表：新增「描述」列 + 垃圾桶删除图标

## 需求内容

- 「参数」等键值表使用带表头的表格，表头为 **名称 | 值 | 描述 | （删除）**；保留最左的
  「启用/停用」勾选列（可临时禁用某行、从请求中移除）。
- 删除操作用**垃圾桶图标**替代原来的 `×`，美观简洁、与整体协调。
- 经确认：**描述列推广**到所有报文键值表（参数 / 请求头 / 表单），垃圾桶删除在**全部**
  行删除处统一（含断言表）。

## 执行计划

### 数据模型（`case.ts`）

- `KV` 接口新增可选 `description?: string`。
- `normalizeKV` 读入 `description`（非空才留）；`serializeKV` 在 `description` 非空时落盘
  （空则省略，不影响实际请求）。
- YAML 中键值项形如 `{ name, value, enabled?, description? }`。

### 请求编辑器（`RequestEditor.tsx`）

- 新增 `TrashIcon`（线条描边垃圾桶，`currentColor`，hover 由 `.row-del` 变红）。
- `KVTable` 加 `withDescription` 开关：为真时表头多「描述」列、每行多描述输入；末行追加空行的
  判定纳入 `description`；表头「键」改为「名称」。删除按钮改用 `TrashIcon`。
- 调用点：参数 / 请求头 / 表单 传 `withDescription`；**outputs、环境变量不传**
  （其数据模型为 `RequestOutput[]` / `Record`，存不下 description）——但仍统一用垃圾桶删除。
- `AssertTable` 的行删除也改用 `TrashIcon`，保持全局一致。

### 样式（`App.css`）

- `.row-del` 改为 `inline-flex` 居中承载 SVG（去掉原 `×` 的字号）；新增 `.trash-ico`。
- hover 变红沿用 `var(--red)`；深浅色主题自动适配。
- **Excel 风格网格**：`KVTable` 的 `<table>` 加 `grid` 修饰类，`.kv-table.grid` 用 `border-collapse`
  合并出连续单线网格，单元格 `padding:0`、输入框内嵌填满（去边框圆角），聚焦时 `outline`（内缩）
  高亮所在单元格。仅作用于键值表；断言表（含 Select 下拉）与只读响应头表不加 `grid` 类、保持原样。

### 数据流校验

- `requestToDraft` / `draftToRequest` 透传 `KV[]`，description 随参数表编辑持久化。
- `onQueryChange`（表格编辑）保留 description；直接改 URL 文本会按 URL 重建 query（description
  与 enabled 一样重置）——沿用既有「URL 为源」的取舍，未改。
- 发送：`buildApiRequest` 只取 name/value，description 不进请求。
