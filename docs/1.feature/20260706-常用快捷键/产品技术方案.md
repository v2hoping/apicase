# 常用快捷键 + 配置页管理

_需求日期：2026-07-06_

## 需求

- 为常用操作增加键盘快捷键；「查看 + 完整自定义重绑」，覆盖**核心动作**。
- 在配置页（`SettingsPage`）新增「快捷键」分区，支持：查看、录制重绑定、冲突检测、恢复默认、禁用。
- 快捷键是 **app 全局偏好**，持久化到 `localStorage`（区别于工作空间级的 `application.yml`）。

## 动作集（核心 7 个）

| 动作 | 默认（macOS） | 默认（Win/Linux） | id |
|---|---|---|---|
| 新建用例 | ⌘N | Ctrl+N | `new-case` |
| 打开工作空间 | ⌘O | Ctrl+O | `open-workspace` |
| 保存 | ⌘S | Ctrl+S | `save` |
| 关闭当前标签 | ⌘W | Ctrl+W | `close-tab` |
| 搜索用例 | ⌘P | Ctrl+P | `search` |
| 打开配置页 | ⌘, | Ctrl+, | `open-settings` |
| 发送请求 | ⌘Enter | Ctrl+Enter | `send-request` |

## 设计

### 跨平台键位
修饰键统一抽象为 **Mod**：macOS = ⌘，其它 = Ctrl。一套配置跨平台生效；展示时按平台渲染（`⌘⇧⌥` vs `Ctrl+Shift+Alt`）。

### 存储（`localStorage`）
- key：`apicase.shortcuts.v1`，值为 override 映射 `{ [actionId]: accelString }`。
- 语义：**缺省** = 用默认；**空串** = 显式禁用；**非空** = 自定义键位。恢复默认 = 删除该 key。
- 规范化字符串形如 `mod+shift+f` / `mod+enter`，用于比较、存储、反查。

### 匹配 / 分发
- 单一全局 `document` keydown 监听（空依赖 `useEffect` + `ref` 取最新 state/lookup/handlers，复用现有 `saveRef` 模式）。
- 反查表 `canonical → actionId → handler`。
- 纯键（无 Mod/Alt）在 `input/textarea/contenteditable` 中**不拦截**，避免干扰输入；带 Mod 的组合在输入框内也生效。

### 配置页 UI
- `SettingsPage` 的 NAV 增加「快捷键」；按分组（文件 / 导航 / 运行）列出：动作名 + 当前键位徽章 + 「修改（录制）」/「恢复默认」；顶部「全部恢复默认」。
- 录制：`document` capture 阶段 keydown + `stopPropagation`（录制时不误触发动作）；`Esc` 取消，`Backspace/Delete` 清空为禁用。
- 冲突：`findConflict` 检测，`confirm` 后替换（占用项自动解绑）。
- `open-settings`（⌘,）直接打开配置页并切到可视。

## 实现

- 新增 `app/src/shortcuts.ts`：动作注册表 + `Accel` 解析/格式化/持久化/冲突检测（纯逻辑，无 React）。
- 改 `app/src/App.tsx`：
  - `SettingsPage` 加「快捷键」分区；新增 `ShortcutsSettings` 组件（录制 + 列表）。
  - `App()` 加 `scOverrides` state、全局分发 `useEffect`（**替换**原独立的 Cmd/Ctrl+S 监听）、`searchInputRef`、各动作 handler 的 ref。
- 少量 CSS（`App.css`）。
