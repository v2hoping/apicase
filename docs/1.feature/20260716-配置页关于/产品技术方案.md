# 配置页「关于」分区

## 需求内容

配置页（`application.yml` 可视化设置页）左侧菜单栏新增**「关于」**项，并实现其内容：
向用户展示应用的元信息，作为软件的「关于本应用」入口。

「关于」页包含：

- **鹦鹉螺 Logo + 名称 + 版本**：复用项目既有的鹦鹉螺图标（`/nautilus.svg`）；应用名 `Apicase`、版本 `0.1.0`。
- **一句话简介**（精炼）：API 接口调试 / 管理 / 用例编排，用例建模为 DAG，文件即数据（local-first）。
- **系统信息**：操作系统（名称 + 版本）、芯片（型号 + 架构）。
- **外链**：项目主页 / 源码（GitHub），点击用系统默认浏览器打开。
- **版权**：© 2026 Apicase。

> 迭代调整：去掉「参考理念（Bruno/Postman）」致谢与「技术栈」表格；图标由自绘节点标记改为项目既有鹦鹉螺；简介进一步精炼；新增系统信息展示。

## 执行计划

1. `SettingsPage` 的 `NAV` 数组追加 `"关于"`（位于「快捷键」之后），渲染 `<AboutSettings />`。
2. 前端 `AboutSettings` 组件：鹦鹉螺 Logo（`<img src="/nautilus.svg">`）+ 名称/版本 + 精炼简介 + 系统信息表 + GitHub 外链 + 版权。
3. 元信息用常量 `APP_VERSION` / `APP_REPO`，与 `package.json`、`tauri.conf.json`、git remote 保持一致。
4. **系统信息经后端获取**：新增 Rust 命令 `system_info`（返回 `os` / `arch` / `chip`）。
   - macOS：`sw_vers -productVersion` 取系统版本、`sysctl -n machdep.cpu.brand_string` 取 CPU 品牌串（如 `Apple M1`）。
   - 其它平台：`std::env::consts::OS` / `ARCH` 退回。
   - 前端 `useEffect` 内 `invoke("system_info")` 拉取，未取到时占位 `—`。
5. 外链通过 `@tauri-apps/plugin-opener` 的 `openUrl` 由系统浏览器打开（`capabilities/default.json` 已授权 `opener:default`）。
6. 样式沿用既有 `settings-*` 版式与主题 CSS 变量，新增 `.about-*`（`about-logo` / `about-sys*` / `about-link` 等）。

已落地（2026-07-16）。前端 `tsc --noEmit`、后端 `cargo check` 均通过；本机预览系统信息正常（macOS 14.6.1 / Apple M1 / arm64）。
