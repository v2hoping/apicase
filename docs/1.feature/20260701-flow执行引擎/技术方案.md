# flow 执行引擎（变量透传 + 输出提取 + 断言）

> 需求时间：2026-07-01。在「flow DAG 画布与三视图」之上，让 flow **真正跑起来**：跨步变量透传（`{{steps.x.outputs.y}}`）、响应 JSONPath 输出提取、响应断言。此前「运行全部」只是逐节点发请求，数据不流动。

## 需求内容

1. **变量透传**：运行时把 case 级 `vars` 与上游步骤已提取的 `outputs` 注入请求各字段（url / query / headers / auth / body）。语法沿用 `{{ }}`：`{{baseUrl}}`、`{{steps.login.outputs.token}}`。未解析的表达式**保留字面量**，便于发现问题。
2. **输出提取**：步骤响应体按 `outputs`（JSONPath）提取为变量，供下游引用。
3. **断言**：每步（及单节点顶层）可配置断言，运行后逐条 ✓/✗；任一断言失败该节点标红。断言失败纳入节点运行状态。

## Schema 增量（`case.ts`）

- 新增 `Assertion { target, op, value? }` 与 `AssertOp`（`eq|ne|contains|exists|notExists|gt|lt|matches`，借 Step CI check / Bruno assert 收敛形）。
- `Step` 增 `assertions: Assertion[]`；`Case` 增 `assertions?`（单节点顶层）。
- 解析/序列化：flow 步级 `assertions:`、单节点顶层 `assertions:`；`exists/notExists` 不落 `value`；空断言不落盘。
- `target` 取值：`status` | `header.<名>`（大小写不敏感）| JSONPath（`$.data.token`）。

## 执行引擎（`flow.ts`，纯前端）

后端仍只做 `send_request`；编排语义在前端：

- `resolveString / resolveDraft`：`{{ }}` 替换。`steps.<id>.outputs.<name>` 查上游 outputs，其余查 `vars`（支持 `vars.` 前缀）。
- `extractJsonPath`：JSONPath **常用子集**（`$` / `.key` / `[n]` / `['key']` / `["key"]`，允许省略前导 `$`/`.`）；未识别片段视为无效路径返回 `undefined`。（通配符/过滤器等留后续）
- `extractOutputs`：按 outputs 从响应体提取。
- `evalAssertions`：逐条评估，数值/字符串宽松比较（`looseEq`），返回 `{target, op, ok, actual}`。

## 编排（`App.tsx`）

- `runStepWithCtx(sd, ctx)`：**变量透传 → send_request → 提取 outputs → 评估断言**；节点状态 = HTTP 成功且断言全过才 `ok`。
- **跑此步**：用当前 `outputsCtx`（含此前各步 outputs）作上下文；跑完回填该步 outputs。
- **运行全部**：拓扑序执行，用**本地上下文对象**在 `await` 之间同步透传 outputs（不依赖异步 state），逐节点回填状态。
- `outputsCtx` 在打开/关闭 case 时重置；「运行全部」开始时清空。

## UI（`RequestEditor.tsx`）

- 请求编辑器新增两个 Tab：**Outputs**（仅 flow，变量名 / JSONPath）与 **Assert**（单/多节点通用）。
- Assert 表：目标 / 操作符 / 期望值；运行后每行 ✓/✗（hover 看实际值）；Assert Tab 角标显示 `通过/总数`（全过绿、有失败红）。
- `StepMeta` 精简为 ID + 依赖（输出移入 Outputs Tab）。
