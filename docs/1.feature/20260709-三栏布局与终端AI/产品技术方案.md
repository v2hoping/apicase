# 三栏布局：左文件树 / 底部终端 / 右侧 AI 对话

> 需求日期：2026-07-09

## 一、需求内容

参考 VSCode 的编辑器布局，在顶栏**最右侧**新增三个切换按钮，分别显隐三块面板：

| 按钮 | 面板 | 位置 | 内容 |
| --- | --- | --- | --- |
| 切换左侧边栏 | 文件树（已有） | 左 | 工作空间文件树 |
| 切换底部栏 | 终端 | 下 | **真·交互式终端**，功能效果与 VSCode 内置终端一致 |
| 切换右侧边栏 | AI 对话 | 右 | 对话界面（本期先搭 UI 外壳） |

### 交互约定

- 三个切换按钮为 VSCode 风格图标（外框方块 + 对应边高亮），放在顶栏最右（`margin-left:auto`）。
- 默认显隐：左侧文件树 **开**，底部终端与右侧 AI **关**。
- 显隐状态持久化到 `localStorage`（`apicase.layout.v1`），下次启动记住。
- 底部栏、右侧栏均可拖动调整尺寸（底部调高、右侧调宽），复用既有 resizer 模式。
- 快捷键（并入既有快捷键注册表，分组「视图」）：
  - 切换左栏：`Mod+B`
  - 切换底部栏：`Mod+J`
  - 切换右栏：`Mod+Alt+B`

## 二、技术方案

### 布局结构

```
.app
├── .topbar            工作空间/环境/配置 … + .layout-toggles（最右三按钮）
└── .body-layout
    ├── .sidebar        文件树（showLeft 控制）
    ├── .sidebar-resizer
    ├── .center-col     纵向：主工作区 + 底部终端
    │   ├── .workspace  （原主区）
    │   ├── .panel-resizer-h  底部栏高度拖拽条
    │   └── .bottom-panel     终端（showBottom 控制）
    ├── .right-resizer
    └── .ai-panel       AI 对话（showRight 控制）
```

### 终端（真 PTY）

- **后端（Rust / `portable-pty`）**：托管状态 `PtyState = Mutex<HashMap<id, PtySession>>`。命令：
  - `terminal_open(id, cwd, cols, rows)`：创建 PTY 对，按 `$SHELL`（缺省 `/bin/zsh`/`bash`）在 `cwd` 起 shell；后台线程读 master 输出，按 `terminal://data/{id}` 事件把原始字节（`Vec<u8>`）发往前端；EOF 发 `terminal://exit/{id}`。
  - `terminal_write(id, data)` / `terminal_resize(id, cols, rows)` / `terminal_close(id)`。
- **前端（`xterm.js` + `addon-fit`）**：`TerminalPane` 组件挂载 xterm，`FitAddon` 自适应；`onData` → `terminal_write`；监听数据事件 → `term.write(Uint8Array)`（原始字节交由 xterm 缓冲，天然处理跨包 UTF-8）；`ResizeObserver` → fit + `terminal_resize`。**首次打开后常驻**（隐藏用 `display:none`），切回时 refit + focus，保持会话与滚动。

### AI 对话（本期 UI 外壳）

- `AiChat` 组件：消息气泡列表（可滚动、自动到底）+ 多行输入（Enter 发送 / Shift+Enter 换行）+ 发送/清空。
- 发送后追加用户消息与一条占位助手回复；状态自持有，面板首次打开后常驻，保留历史。
- 后续接入真实模型时，仅需替换「发送」的回复来源。
