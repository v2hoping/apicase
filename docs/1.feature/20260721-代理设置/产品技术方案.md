# 代理设置（是否使用系统代理）

## 需求内容

apicase 后端用 `reqwest` 发请求，默认会读取系统 `HTTP_PROXY` / `HTTPS_PROXY` / `ALL_PROXY` 环境变量。当从带代理环境的终端启动应用、或系统开了全局代理时，请求**本地地址**（`127.0.0.1` / `localhost`）会被代理拦截返回 **502**，无法调试本地服务（如本地 httpbin）。

需在「设置」中增加「代理」分区，让用户控制发请求时的代理行为：

- **跟随系统**（默认，保持原行为）——读系统代理环境变量。
- **不使用代理**（直连）——忽略系统代理，解决本地调试被拦。
- **自定义**——指定 http / https 代理地址。

## 执行计划

### 后端（`app/src-tauri/src/lib.rs`）

- 新增 `ProxyConfig { mode: String, url: Option<String> }`（serde camelCase）。
- `perform_request` 增参 `proxy: Option<ProxyConfig>`，按 `mode` 构建 client：
  - `none` → `client_builder.no_proxy()`
  - `custom` → `Proxy::all(url)`（url 为空则退化为 `no_proxy()`，地址非法则返回错误）
  - `system` / 缺省 → 保留 reqwest 默认（读系统代理环境变量）
- `send_request` 命令透传 `proxy`；三处单元测试调用改为 `perform_request(req, None)`。
- reqwest 当前仅 `rustls-tls` feature，`custom` 支持 **http/https** 代理；socks 需另加 `socks` feature（本期不启用）。

### 前端

- 新增 `app/src/proxy.ts`：`ProxyMode`/`ProxyConfig` 类型 + `loadProxyConfig`/`saveProxyConfig`（localStorage `apicase.proxy.v1`）+ `proxyPayload`（发送给后端的载荷，url 空时省略）。
- `App` 持有 `proxyConfig` state；发请求（`invoke("send_request", …)`）时带上 `proxy` 参数。
- `SettingsPage` 新增「代理」分区（跟随**应用**组，位于「主题」之后）：三张模式卡片（跟随系统 / 不使用代理 / 自定义）+ custom 模式下的地址输入框；导航新增「代理」图标（地球）。
