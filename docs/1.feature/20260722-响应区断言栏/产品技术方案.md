# 响应区「断言」栏

_创建：2026-07-22_

## 需求

响应区原有「响应体 / 响应头」两个 Tab。在「响应头」右侧新增一个「断言」Tab，
把本次运行的**断言结果**集中展示在这里，与响应体/响应头并列查看。

此前断言结果仅在**请求编辑器**的「断言」Tab 中逐条 ✓/✗ 呈现；调试时想同时看
响应内容与断言结论需来回切换。现将结果同步进响应区，形成「看响应即看断言」的闭环。

**配置/结果分离**：结果迁入响应区后，请求编辑器的「断言」Tab **只负责配置**，
不再显示运行结果——去掉配置表的 ✓/✗ 结果列，Tab 徽标改为纯配置数量。
结果的唯一展示位即响应区「断言」栏。

## 方案

- **数据来源**：复用运行态 `run.asserts`（`AssertResult[]`：`target/op/ok/actual`），
  由 `evalAssertions` 在发送后计算，已过滤空目标，按序对齐已填断言；
  「期望值」列从 `selected.assertions`（同样过滤空目标）按下标补齐。
- **Tab 类型**：`respTab` 联合类型由 `"body" | "headers"` 扩为
  `"body" | "headers" | "assert"`（含 `TabSnapshot` 快照字段）。
- **Tab 徽标**：「断言」标签右上角显示 `通过/总数` 上标（复用 `.tab-count`
  的 `all-ok`/`has-bad` 绿/红语义），无结果时不显示。
- **结果表**：只读表（复用 `.kv-table.readonly`）——列为
  `结果(✓/✗) | 目标 | 断言 | 期望值 | 实际`；`exists/notExists` 无期望值显示 `—`；
  操作符经 `OP_LABELS` 显示中文（由 `RequestEditor` 导出复用）。
  无断言时显示「未设置断言」。

## 落地文件

- `app/src/RequestEditor.tsx`：`OP_LABELS` 改为 `export`；`AssertTable` 移除
  ✓/✗ 结果列与 `results` 属性；断言 Tab 徽标并入通用 `tabBadge`（纯配置数量）；
  移除 `assertResults` 属性、`assertPass` 与 `AssertResult` 依赖；提示文案指向响应区。
- `app/src/App.tsx`：导入 `AssertOp`/`OP_LABELS`；扩 `respTab` 类型；
  计算 `assertResults/assertPass/assertDefs`；新增「断言」Tab 与结果表；
  不再向 `RequestEditor` 透传 `run.asserts`。
- `app/src/App.css`：新增 `.assert-result-table` 列样式；移除失效的 `.assert-table .res-col`。

## 后续调整：响应区标题栏合并为单行（2026-07-22）

Tab 移入响应区标题栏后，进一步把「响应」标题栏与 Tab 行**合并为一行**：

- **去除**「响应」文字（`.response-title`）与「美化」开关（`.resp-tab-right`/`.pretty-toggle`
  + `pretty` 状态/快照字段）；响应体恒用 `prettyBody` 美化（非 JSON 回退原文）。
- **单行布局**：折叠钮 + 响应体/响应头/断言 Tabs 靠左（`.resp-tabs`），
  状态/耗时/大小靠右（`.response-head-meta`）。整行仍兼作拖动调高手柄与折叠展开；
  Tab `onMouseDown` 阻止冒泡避免误触拖动，点击切 Tab 并顺带展开。
- CSS：`.response-head` 改 `align-items:stretch`（Tab 贴底下划线）+ 展开态 `border-bottom`；
  折叠钮/meta `align-self:center`；移除 `.tabs.sub` 相关样式。
