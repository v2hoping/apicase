# 多标签页 + 视图切换 bug 修复

> 需求时间：2026-07-01。① 修复三视图切换在「关请求→关流程→落文本后，再点流程无响应」的状态 bug；② 引入多标签页（打开多个 case，右键关闭当前/其他/全部）。

## 一、视图切换 bug 修复

**现象**：流程+请求都开时，先关请求（只剩流程）、再关流程 → 自动落文本；此后点「流程」无法切回。

**根因**：旧 `applyPanes(!showFlow, showRequest)` 在两栏都关时只调 `enterText()` 直接 return，**没把 `showFlow` 同步为 false**。于是 `showFlow` 仍为 true，再点流程算出 `!showFlow=false` → 又两栏皆假 → 再次落文本，死循环。

**修复**：改为**意图式**处理（`onClickFlow`/`onClickRequest`）：
- 若当前是文本态 → 退出文本并**确保显示该栏**（`setShowFlow(true)`，文本模式先 `commitText` 提交回结构）。
- 若该栏是当前唯一在显的栏 → 关它 = 落文本（同时把该栏置 false，状态不再陈旧）。
- 否则正常 toggle。

## 二、多标签页

**模型**：活动标签用现有 live state；非活动标签用 `tabCacheRef`（`Record<path, TabSnapshot>`）保存完整编辑态快照。`tabOrder: string[]` 是标签顺序。

- **快照/恢复**：`snapshotCurrent()` 把活动标签的全部 per-case 状态（steps/视图/rawText/runMap/outputsCtx/dirty…）打包；`restoreSnapshot()` 逐一还原。切换标签 = 快照当前 + 恢复目标（新标签则 `openCase` 从磁盘加载）。
- **打开**：文件树点击、新建 case、编辑环境等统一走 `openTab(path)`（已开则恢复内存态、未开则加载）。
- **关闭**：`closeTab`（中键 / × / 右键菜单）关活动标签时激活相邻标签；`closeOtherTabs` / `closeAllTabs`；dirty 标签关闭前 `confirm`。
- **同步**：重命名/删除文件时同步更新 `tabOrder` 与快照 key；切换工作空间清空所有标签。
- **UI**：`TabBar`（文件名 + dirty 点 / hover 显 ×，活动标签白底 + 顶部橙条）；右键弹「关闭当前/其他/全部标签页」。case 顶栏移除文件名（已在标签显示），仅留视图切换 + 保存。

## 涉及文件
