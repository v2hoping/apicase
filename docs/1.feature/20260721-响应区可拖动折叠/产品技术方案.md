# 响应区可拖动折叠

## 需求内容

在「请求」视图里，请求编辑器（`req-editor`）与响应区（`response`）当前是上下自然堆叠、整块一起滚动，两者高度不可调。

本需求让二者之间可**上下拖动**分配高度，并支持把响应区**折叠成一行**：

1. 请求编辑器（上）与响应区（下）之间新增一条可拖动的分界，向上拖增大响应区、向下拖缩小响应区。
2. 拖到最下段（响应区高度低于阈值）时，响应区**自动收起**，只保留一行「响应」标题。
3. 折叠后点击该标题行（或标题左侧三角）即可**重新展开**，恢复到折叠前的高度。
4. 折叠态下，若已有响应结果，标题行右侧仍展示状态码 / 耗时 / 大小，一眼可见。

## 执行计划

### 布局改造（`request-pane`）

- `request-pane` 由 `overflow: auto`（整块滚动）改为 `overflow: hidden` 的上下两段：
  - 上段 `req-scroll`：包裹 `RequestEditor`，`flex: 1` 填充剩余高度、内部滚动。
  - 下段 `response`：高度由状态 `respHeight`（px）控制，`flex: 0 0 auto`。
- 响应区新增始终显示的标题栏 `response-head`，兼作拖动手柄；折叠时仅剩此行。
- 响应内容 `response-content`（tabs + 响应体/响应头）在展开态填充剩余高度、内部滚动。

### 交互逻辑（`App.tsx`）

- 新增状态：`respHeight`（默认 240）、`respCollapsed`、`respResizingRef`、`requestPaneRef`。
- 拖动：在 `response-head` 上 `mousedown` 开始，全局 `mousemove` 用「请求面板底边 − 鼠标 Y」反推目标高度：
  - 高度 < 60px → `respCollapsed = true`（收起为一行）。
  - 否则钳制到 `[120, 面板高 − 150]`，保证请求编辑器至少留 150px。
- 折叠/展开：标题栏左侧三角按钮 + 折叠态整行点击切换 `respCollapsed`；展开恢复到 `respHeight`。
- 拖动风格复用既有 `resizing-row`（`row-resize` 光标 + 全局禁选），与底部终端栏拖高一致。

### 样式（`App.css`）

- 改写 `.request-pane` / `.response` / `.response-body`；新增 `.req-scroll`、`.response-head`、`.response-toggle`、`.response-title`、`.response-head-meta`、`.response-content`。
