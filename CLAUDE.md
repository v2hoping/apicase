# CLAUDE.md

本文件是 apicase 项目的**全局提示词**，用于指导 Claude 在本仓库中的工作方式。

## 语言

- 始终使用**中文**回复、注释与沟通；技术术语与代码标识符保留原文。

## 目录结构

```
apicase/
├── app/                              # 应用代码
├── docs/                             # 文档
│   ├── 0.latest/                     # 当前全局最新的【全部文档】—— 唯一事实来源
│   └── 1.feature/                    # 新需求（每个需求一个子目录）
│       └── 20260629-湖仓一体/        # 子目录命名：YYYYMMDD-需求名
│           └── 产品技术方案.md           # 描述该需求与执行计划
└── CLAUDE.md                         # 全局提示词（本文件）
```

## 文档体系约定

- **`docs/0.latest/`**：项目当前全局最新的全部文档集合，是唯一事实来源（Single Source of Truth）。
  涉及项目现状的任何判断，都应以此目录为准。
- **`docs/1.feature/`**：新需求目录。开发一个新需求时，在其中创建一个以
  `YYYYMMDD-需求名` 命名的子目录（如 `20260629-湖仓一体`），子目录内放一份
  `技术方案.md`，用于描述该需求的**需求内容**与**执行计划**。

## 工作流

1. 开始任何任务前，先阅读 [docs/0.latest/](docs/0.latest/) 了解项目最新全貌。
2. 开发新需求时，在 [docs/1.feature/](docs/1.feature/) 下创建子目录
   `YYYYMMDD-需求名`（如 `20260629-湖仓一体`），并在其中编写 `技术方案.md`
   （需求内容 + 执行计划）。
3. 需求开发完成并落地后，**务必**将变更要点同步合并进 `docs/0.latest/`，
   保持该目录始终反映最新状态。
